# Doma Forge Grant Scope

## Project

**DomainFi Agent Toolkit** — AI-powered agents and developer templates for discovering, monitoring, and acting on tokenized domain opportunities on Doma Protocol.

## Problem

DomainFi has strong potential because domains already have mainstream demand and clear user intent. The friction is that domain markets produce fragmented data: listings, pricing, renewals, expiry windows, ownership state, category relevance, and marketplace movement.

Users need tools that turn that data into actions.

## Proposed contribution

Build an agent layer that helps users and developers:

- discover relevant tokenized domains
- monitor portfolios and watchlists
- receive useful alerts
- understand why a domain is relevant
- copy starter templates for Doma-based bots and dashboards

## Initial deliverables

1. Public architecture and integration plan
2. Domain discovery prototype
3. Watchlist/portfolio alert prototype
4. Telegram/Discord notification examples
5. Developer templates and documentation
6. Public write-up showing how to build agentic DomainFi workflows on Doma
7. Arc/Circle paid-agent MVP showing x402-style pay-per-scan monetization, Arc Testnet constants, and USDC unit economics

## Success criteria

- A developer can clone the repo and understand how to build a Doma-powered domain agent.
- A user can define a watchlist and receive useful alerts.
- The scoring pipeline produces transparent explanations rather than opaque recommendations.
- The project avoids custody, private-key handling, or automated transactions in the first milestone.
- A builder can run a local Arc paid-agent demo and understand how to replace the local x402 test proof with Circle Gateway/x402 verification.
