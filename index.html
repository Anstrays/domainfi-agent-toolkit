Three useful workflows first.

No fake dashboard theater. The first prototype focuses on actions that real DomainFi users and developers need repeatedly.

01
Domain discovery agent

Scans domain inventory and marketplace data, then ranks opportunities by category, relevance, brandability, pricing, and liquidity signals.

02
Portfolio and alerts agent

Tracks owned domains, watchlists, status changes, expiry events, renewal reminders, pricing changes, and marketplace activity.

03
Developer templates

Starter bots, API wrappers, and workflow examples for builders integrating Doma data into dashboards, wallets, marketplaces, and agents.

Doma as the domain layer. Agents as the action layer.

The toolkit is designed to plug into Doma SDKs, APIs, marketplace data, and onchain domain primitives as they become available.

Data access

Query Doma inventory, ownership state, domain status, marketplace listings, and event surfaces.

Scoring pipeline

Normalize domain data into ranked opportunities with explainable filters and user-defined strategies.

Notification adapters

Deliver alerts through Telegram, Discord, CLI, webhooks, or future wallet-facing interfaces.

Milestone-based build plan.

A compact path from research to working public demo.

WEEK 1

Architecture

Map Doma docs, SDKs, APIs, data model, and agent workflow boundaries.

WEEK 2–3

Prototype

Build discovery CLI or bot with filters, watchlists, ranked output, and explanations.

WEEK 4

Alerts

Add Telegram/Discord notification examples and portfolio/watchlist monitoring flow.

WEEK 5–6

Public demo

Ship documentation, templates, and a write-up on agentic DomainFi workflows.
